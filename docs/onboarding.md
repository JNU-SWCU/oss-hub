# 온보딩

oss-hub에 처음 참여해 첫 티켓을 시작하는 팀원을 위한 안내다. 아래 순서만 따라가면 추가 질문 없이 작업을 시작할 수 있다.

## 0. 에이전트 준비 (최초 1회)

작업을 시작하기 전에 에이전트에 craft-skills 스킬 30종을 설치한다. 한 번만 하면 되고, 이후 티켓마다 다시 할 필요는 없다.

- **Claude Code**: repo를 clone한 뒤 그 폴더에서 프로젝트를 연다. 마켓플레이스 신뢰 프롬프트가 1회 뜨는데, 수락하면 craft-skills 30종이 전부 활성화된다.
- **Codex**: 전역에서 한 번만 아래 두 명령을 실행한다.

  ```
  codex plugin marketplace add GoBeromsu/craft-skills
  codex plugin add craft-skills@craft-skills
  ```

## 1. 보드에서 내 티켓 찾기

[프로젝트 보드](https://github.com/orgs/JNU-SWCU/projects/1)를 연다. 담당자(assignee)가 본인으로 지정된 티켓 카드를 찾는다. 카드를 열면 그 작업에 필요한 내용이 티켓 본문에 전부 적혀 있다.

## 2. 에이전트에 시작 프롬프트 붙여넣기

본인이 쓰는 에이전트(Claude Code 등)에 아래 한 줄을 그대로 붙여넣는다. `<번호>` 자리에는 1단계에서 찾은 티켓 번호를 넣는다.

```
oss-hub 티켓 #<번호> 진행해줘
```

작업 중 테스트 계정 등 자격증명이 필요하면 팀 Notion에서 확인한다.

## 3. 브랜치 → 커밋 → PR → In Review

에이전트가 브랜치를 만들고 커밋하고 PR을 연다. 브랜치명·커밋 형식·PR 본문·보안 규칙은 이 문서가 아니라 [AGENTS.md](../AGENTS.md)와 [docs/rules/security.md](rules/security.md)를 따른다 — 궁금하면 그 문서를 확인한다. PR이 열리면 보드에서 해당 카드를 In Review로 옮긴다.

## 역할별 테스트 계정 로그인

화면 작업을 확인하려면 역할(학생·교직원·관리자)별 테스트 계정으로 로그인해야 한다. 계정 자격증명과 역할 매핑 값은 팀 Notion에서 확인한다 — 이 문서와 코드에는 값을 적지 않는다.

로컬에서는 로그인한 GitHub 계정의 역할을 `AUTH_TEST_ROLE_MAP` 환경변수(non-production 전용)로 지정한다. 변수 형식과 실값은 팀 Notion에서 확인한다.

## 티켓 본문 읽는 법

티켓마다 섹션 구성은 조금씩 다를 수 있다. 작업을 시작하기 전에 "최소 요구 (기능)" · "완료 조건 (기능 검증)" · "절대 금지 (이 티켓의 경계)" 세 섹션을 먼저 확인하고, 작업이 이 범위를 벗어나지 않게 한다. 나머지 섹션은 작업의 배경을 이해하는 데 참고한다.
