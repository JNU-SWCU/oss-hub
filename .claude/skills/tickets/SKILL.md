# tickets

oss-hub의 GitHub Issue 티켓을 범위를 벗어나지 않는 검증된 PR로 바꾸는 워크플로다.
성공 기준: PR이 티켓 계약이 요구한 것과 정확히 일치하고, 완료 조건의 모든 항목이 PR을 열기 전에 실증되며, 절대 금지 경계 밖의 어떤 파일도 건드리지 않는다.

## 워크플로

1. 티켓 수집: `gh issue view <번호>` (이 repo 안에서 실행하면 `--repo` 지정은 불필요하다).
2. 계약 파싱 — 아래 세 섹션이 작업 범위의 유일한 원본이다.
   - 최소 요구 (기능) — 구현해야 할 최소 범위. 그 이상도 이하도 만들지 않는다.
   - 완료 조건 (기능 검증) — PR을 열기 전 실제로 증명해야 하는 체크리스트.
   - 절대 금지 (이 티켓의 경계) — 이 티켓이 건드리면 안 되는 파일·동작.
   선행 의존성(선행 조건) 섹션이 미충족 의존성을 명시하면, 부분 작업을 시작하지 않고 그 사실을 그대로 보고한다.
3. 코드를 쓰기 전에 [이 repo의 AGENTS.md](../../../AGENTS.md)와 관련 경로의 중첩 AGENTS.md(예: `apps/AGENTS.md`, `apps/frontend/AGENTS.md`, `apps/backend/AGENTS.md`)를 읽는다. 이 스킬은 브랜치·커밋·PR 흐름이나 보안 규칙을 복제하지 않는다 — 원본과 스킬 사본이 어긋나는 drift를 막기 위해서다. 실제 규칙은 항상 AGENTS.md와 그 링크 문서([docs/rules/security.md](../../../docs/rules/security.md), [docs/rules/pr-scope.md](../../../docs/rules/pr-scope.md))에서 확인한다.
4. 최소 요구 섹션이 요구하는 것만 구현한다. 지금 같이 고치면 편해 보이는 인접 기능이나 스키마 변경도 그 자체의 티켓 몫으로 남긴다.
5. 절대 금지 섹션을 문자 그대로 지킨다 — 그 섹션이 이름을 언급한 파일·영역은 0건 수정한다.
6. 완료 조건 체크리스트를 한 항목씩 실제로 구동해 증명한 뒤에만 체크한다(테스트 실행, 화면·플로우 직접 확인 등). 검증하지 않은 체크리스트로 PR을 열지 않는다.
7. PR을 열기 전 `bash scripts/check-public-safe.sh`로 변경 파일·커밋 메시지의 public-safe 위반(실명·전화번호·개인 머신 경로 등, [docs/rules/security.md](../../../docs/rules/security.md) deny-list)을 사전 검사한다. 이 repo는 PUBLIC이므로 PR 본문·코멘트에도 같은 기준을 적용한다.
8. [AGENTS.md](../../../AGENTS.md)가 정한 흐름대로 PR을 연다 — 브랜치명·Conventional Commits·PR 본문 형식은 AGENTS.md §5가 원본이다.
9. PR이 열리면 [프로젝트 보드](https://github.com/orgs/JNU-SWCU/projects/1)에서 해당 티켓 카드를 In Review로 옮긴다.

## 이스케이프 해치

- 선행 의존성이 미충족 → 어떤 의존성인지 정확히 보고하고 멈춘다. 부분 구현을 시작하지 않는다.
- 계약이 구현에 필요한 지점에서 침묵하거나 모호함 → 무관한 문서·코드 주석·비슷한 과거 티켓의 기억으로 추론하지 않고 사용자에게 묻는다. 티켓 본문은 그 자체로 완결된 설계로 간주한다.

## 요구사항

- `gh` — issue view, PR 생성, 프로젝트 보드 카드 이동.
- `git` — 브랜치·커밋 메커니즘은 이 repo의 [AGENTS.md](../../../AGENTS.md)를 따른다.
- `bash scripts/check-public-safe.sh` — PUBLIC repo 텍스트 안전 사전 검사.

## 안티패턴

- 최소 요구 섹션이 요구하지 않은 "개선"을 얹는다 → 이 PR에서 빼고 별도 티켓으로 분리하거나 제안한다.
- 계약에 없는 요구사항을 외부 문서·코드 주석·기억에 의존해 추론한다 → 티켓 본문만이 유일한 원본이며, 침묵하면 사용자에게 묻는다.
- 실제로 구동해보지 않고 완료 조건 항목을 체크한다 → 먼저 테스트나 플로우를 구동한 뒤에 체크한다.
- 이 repo의 AGENTS.md 규칙을 스킬 안에 다시 옮겨 적는다 → 원본과 어긋나는 drift가 생긴다. 항상 AGENTS.md를 직접 확인한다.

## 검증

- [ ] PR을 열기 전에 최소 요구·완료 조건·절대 금지 세 섹션을 읽었다.
- [ ] 선행 의존성이 있다면 확인했고, 미충족이면 작업을 시작하지 않고 보고했다.
- [ ] 이 repo의 AGENTS.md(루트 + 관련 중첩 AGENTS.md)를 첫 수정 전에 읽었다.
- [ ] 절대 금지 섹션 밖의 파일은 0건 수정했다.
- [ ] 완료 조건 각 항목을 실제 증거로 실증했다(주장만으로 체크하지 않았다).
- [ ] `bash scripts/check-public-safe.sh`를 PR 전에 실행했다.
- [ ] AGENTS.md가 정한 흐름대로 PR을 열었고, 프로젝트 보드 카드를 In Review로 옮겼다.
